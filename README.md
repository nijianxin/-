# -
1.Universal GUI Interaction Research Platform
面向“行为语义层”的通用交互研究框架
1. 项目本质（What this project actually is）

本项目并非游戏 AI、自动化脚本或视觉点击工具。

它是一个研究平台，用于回答一个更基础的问题：

计算机是否可以在不依赖具体界面结构的情况下执行任务？

我们尝试构建一种 界面无关（Interface-Independent）的可执行行为语言，
并研究如何将该语言映射到任意图形化程序（GUI）。

换句话说：

人类使用软件时的过程是：

目标 → 理解功能 → 操作界面

而计算机自动化通常是：

识别像素 → 点击坐标

本项目试图建立第三条路径：

目标 → 行为语义 → 映射到界面

GUI 在这里不再是“被识别的对象”，
而是“可实现行为的执行环境”。

2. 为什么需要这个研究（Motivation）

当前自动化与 AI 控制软件的方式存在根本限制：

方法	本质	局限
脚本自动化	依赖控件结构	软件变化即失效
RPA	依赖流程录制	无法泛化
视觉点击 AI	依赖像素特征	不可迁移
多模态模型	识别界面语义	仍依赖界面

它们共同的问题是：

行为绑定在界面上

而人类的能力是：

行为独立于界面
（无论按钮长什么样，你都知道“保存”的意义）

本项目研究的核心命题：

是否存在一种“行为层”，可以作为人机交互的共同中间表示？

3. 核心思想：行为语义层（Action Semantics Layer）

我们提出在 UI 与 AI 之间引入一个中间层：

Human Goal
    ↓
Action Semantics Layer
    ↓
GUI Mapping Layer
    ↓
Actual Software

该层由 可组合的行为原子（Action Primitives） 构成，例如：

行为原子	含义
Select(Object)	选择对象
Create(Entity)	创建实体
Modify(Parameter, Value)	修改参数
Acquire(Resource)	获取资源
Commit()	提交操作
Switch(Context)	切换上下文

AI 不再直接操作界面，而是执行行为语句：

Acquire(Resource)
Create(DefenseUnit, Lane=3)
Commit()

然后系统将其映射为：

点击卡牌 → 点击格子 → 等待动画

4. 三阶段研究路径（Research Phases）
Phase 1 — 闭环存在性验证

目标：证明行为可以被独立描述

环境：单一可控程序（如单机游戏）

构建：

感知 → 状态抽象 → 行为 → 执行 → 状态变化

关键产出不是“通关”，而是：

STATE → ACTION → STATE_CHANGE

记录成为可复用行为数据。

Phase 2 — 行为原子归纳

目标：构建可跨程序复用的行为字典

从多个程序中抽取共性：

不同程序中的操作	抽象行为
放置植物	Create(Entity)
新建文件	Create(Entity)
添加商品	Create(Entity)

此阶段完成时：

AI 已不再学习“软件”，而在学习“行为”。

Phase 3 — 意图驱动执行

目标：从目标直接生成行为序列

用户输入：

“整理库存”

系统流程：

生成行为计划

在界面中寻找实现路径

执行

此时多模态仅用于：

将界面解析为“可执行能力集合”，而非识别像素

5. 与传统 AI 自动化的区别
系统	控制层级
宏脚本	坐标层
RPA	控件层
视觉 AI	图像层
多模态代理	语义 UI 层
本项目	行为层

这是一次层级提升，而非能力提升。

6. 伦理定位

本项目定位为：

人机交互研究

无障碍辅助技术原型

AI 行为表示研究基准

自动化理论验证环境

不面向在线对抗环境或绕过保护机制。

7. 项目一句话

构建一种界面无关的可执行行为语言，并研究其向现实软件的映射方法。

8. 当前状态

Early Research Prototype
正在验证行为语义层的可行性

9. 最终目标

如果成功，该项目将意味着：

软件不再需要被“适配”，
而只需要被“理解为可执行能力集合”。

这可能成为一种新的通用人机交互范式。
